## A. 배포 & 캐시

### A1. "코드 고쳤는데 폰 앱에 반영이 안 돼요"
- **증상**: 로컬에서 다 됐는데 라이브 PWA는 옛날 화면.
- **원인**: ① 커밋만 하고 `git push`를 안 함 → GitHub/Vercel은 옛 버전. ② push해도 PWA가 Service Worker 캐시로 옛 HTML 서빙.
- **교훈**: 반영 안 될 때 **체인을 끝까지 확인**: `디스크 파일 → 커밋 → push → Vercel 배포 → SW 캐시`. 어디서 끊겼는지부터 짚는다. "보는 화면이 file://인지 라이브 URL인지"를 가장 먼저 확정.

### A2. SW 캐시가 끈질기다
- **원인**: `network-first HTML`이라도 PWA standalone은 갱신 타이밍이 느릴 수 있음.
- **교훈**: HTML만 바꿨으면 PWA **완전 종료 후 재실행**이면 보통 픽업. 그래도 안 되면 Safari 웹사이트 데이터 삭제. **`sw.js` 자체를 고쳤을 때만** `CACHE` 버전 상수를 bump한다(콘텐츠만 바꿨는데 bump하면 불필요한 전면 재캐시).

### A3. push는 함부로 하지 않는다
- **교훈**: push = 외부 배포(되돌리기 번거롭고 outward-facing). **오너가 명시 요청할 때만** 실행하고, 평소엔 "푸시 직전까지"만. 커밋까지는 자유, push는 확인 후.

## B. 환경 & 권한 (macOS / Desktop)

### B1. `~/Desktop` 트리의 TCC / `com.apple.macl` 잠금
- **증상**: 파일을 브라우저/에디터로 연 뒤 갑자기 Claude의 파일 read/write가 전부 `Operation not permitted`.
- **원인**: macOS TCC 보호 폴더에서 파일을 GUI 앱이 열면 `com.apple.macl` xattr가 붙어 다른 프로세스 접근 차단.
- **교훈**: 프로젝트는 가능하면 **`~/Develop/` 등 비보호 폴더**에 둔다. Desktop이면: 파일 연 앱 닫기 + Claude 재시작, 또는 터미널에 Full Disk Access 부여.

### B2. Desktop에서 헤드리스 Chrome 금지
- **원인**: 헤드리스 크롬 구동이 위 TCC 잠금을 유발.
- **교훈**: 시각 검증은 **헤드리스 대신** ① 텍스트/구조 검증(grep·구문검사) ② 일반 Safari `open`(오너가 직접 봄)으로 대체.

## C. 코드 & Babel & CSS

### C1. JSX Fragment `<>` 가 안 먹는다
- **원인**: Babel standalone(브라우저 런타임 컴파일)이 일부 fragment 문법 처리 못 함.
- **교훈**: `<div>` 또는 `React.Fragment` 사용. 단일 파일 = Babel standalone 전제이므로 항상.

### C2. 미정의 CSS 변수를 써서 스타일이 조용히 사라짐
- **증상**: `border: 1px dashed var(--color-hairline)` 인데 선이 안 보임(에러도 없음).
- **원인**: 존재하지 않는 토큰명. CSS 변수는 미정의여도 에러 없이 무시됨 → 디버깅 어려움.
- **교훈**: 토큰 쓰기 전 **`grep -- "--토큰명:"`로 정의 존재 확인**. 작업 후 `grep "var(--…)"` 대비 정의 목록 대조.

### C3. SMIL 애니메이션은 살아있다(iOS Safari)
- **교훈**: `<animateMotion>`/`<animate>`(SMIL)은 React JSX에서 그대로 렌더되고 iOS Safari가 지원 → **의존성 0으로 경로 애니메이션** 가능. 단 `prefers-reduced-motion`은 SMIL을 CSS로 못 끄니, draw류는 CSS keyframes로 따로 빼서 reduced 처리.

### C4. 외부 이미지·폰트는 반드시 죽는다고 가정
- **증상**: Unsplash 직링크가 깨져 검정 박스.
- **교훈**: 모든 외부 이미지에 **`onError` → 그래디언트/폴백**. 폰트도 시스템 폰트 폴백. "외부 자원은 언젠가 죽는다"를 기본값으로.

### C5. 토큰 *이름*과 *실제 값*은 다를 수 있다 — 가정 금지
- **증상**: "흰색 하드코딩을 토큰으로 통일"하려고 `#fff` → `var(--color-text-inverse)`로 바꾸면 색이 **크림색**으로 변함.
- **원인**: `--color-text-inverse`가 순백이 아니라 `var(--color-atomic-cream-50)`(따뜻한 크림)였음. 이름만 보고 "inverse=흰색"이라 가정.
- **교훈**: 토큰으로 대체하기 전 **정의값을 grep으로 직접 확인**(`grep -- "--토큰명:"`). "시각 무변화" 교체는 *값이 동일한* 토큰으로만(순백은 `--color-atomic-white: #FFFFFF`). 이름의 의미를 신뢰하지 말 것.

### C6. "토큰화 = 시각 무변화"가 아니다 — 상위 인라인 오버라이드·단위 환산 주의
- **증상**: ① `body { font-family: var(--font-sans) }`(Pretendard)인데 한글 본문이 애플고딕으로 렌더됨. ② 자간 `0.2px`를 `--track-label`(0.08em)로 "정합화"하면 자간이 눈에 띄게 벌어짐.
- **원인**: ① App 루트 컨테이너가 인라인 `fontFamily:'-apple-system,…'`로 body 토큰을 **덮어쓰고** 있었음 → 하위 전체가 상속. ② `0.2px`(≈0)와 `0.08em`(@16px≈1.3px)는 전혀 다른 값.
- **교훈**: "하드코딩 → 토큰"이 항상 무변화는 아니다. **상위에서 같은 속성을 인라인으로 덮고 있는지** 먼저 확인(덮고 있으면 토큰 통일 시 렌더가 *바뀐다* — 보통 의도된 정상화지만 시각 변화이므로 오너 승인·실측 필요). px↔em 환산은 **실제 픽셀로 비교**하고, 무변화가 아니면 "안전 정리"에서 빼고 따로 보고.

### C7. `box-shadow`는 그라데이션이 안 된다 — glossy 띠는 클래스+`::before`
- **증상**: 카드 왼쪽 띠를 금속 광택(다중스톱 그라데이션)으로 하고 싶은데 `boxShadow:'inset 4px 0 0 …'`는 단색만 됨.
- **원인**: box-shadow 색은 단색. 인라인 React는 `::before` 가상요소도 못 만든다.
- **교훈**: 그라데이션·sheen이 필요한 장식 테두리는 **`<style>`에 CSS 클래스 + `::before`**(absolute 4px 띠, `linear-gradient`)로 빼고 JSX엔 `className`만. 모션은 `@media (prefers-reduced-motion: reduce)`로 끈다. 과거 인라인 `border` shorthand+longhand 혼용이 리렌더로 사라진 버그(`80f5438`)도 같은 맥락 — 장식 테두리는 인라인보다 클래스가 안전.

### C8. 동기 `<script>`가 첫 페인트를 막는다 — `defer`로 분리
- **증상**: 로딩 스플래시(브랜드 화면)가 즉시 안 뜨고 스크립트 다운로드(특히 babel-standalone ~324KB)를 기다린 뒤 나타남.
- **원인**: `<head>`의 동기 `<script>`는 HTML 파싱을 블로킹 → `<body>`의 스플래시 마크업이 스크립트 로드 후로 밀림.
- **교훈**: React·ReactDOM·babel·동기화 라이브러리에 **`defer`**를 준다. defer는 문서 순서를 보장해 의존성(React→ReactDOM→babel)이 유지되고, 파싱을 막지 않아 스플래시가 즉시 페인트된다. **babel-standalone은 `DOMContentLoaded`에서 `transformScriptTags()`를 호출하므로 defer와 호환**(defer 스크립트는 DOMContentLoaded 직전 실행돼 리스너가 항상 등록됨). 단 babel 런타임 컴파일이 근본 병목이라 *체감* 개선까지가 한계(빌드 도입 없이는). **흰 화면 리스크가 있으니 실기기 1회 확인 필수.**

### C9. 실시간 동기화는 "편집 중인 입력"을 조용히 덮는다 — 기준값+dirty로 충돌 인지
- **증상**: 두 기기가 같은 메모를 동시에 고치면, 원격 변경이 도착하는 순간 내가 입력 중이던 내용이 사라짐(last-write-wins 손실).
- **원인**: `useEffect(()=>setVal(remote),[remote])`가 원격 값을 **무조건** textarea에 덮어씀.
- **교훈**: 마지막 동기화/저장값을 **`baseRef`(기준값)**에 두고, 원격이 올 때 `dirty = (현재입력 !== 기준값)`을 판별한다. **편집 중이 아니면 그대로 수용**(기존 UX), **편집 중이면 내 입력 보존 + 충돌 배너**([상대 버전으로]/[내 입력 유지]). 자기 저장 에코는 `_lastPushAt`/기준값 일치로 무시. 동시편집 가능한 모든 텍스트 필드에 적용.

### C10. 오버레이 어포던스는 `pointerEvents:'none'`으로 — 기존 탭을 가로채지 않기
- **증상**: "밀 수 있음"을 알리는 chevron을 카드 우측에 얹었더니, 그 영역의 체크 토글·`details` 펼침·맵링크 탭이 chevron에 먹힘(클릭 핸들로 만들면 회귀).
- **교훈**: 순수 시각 신호(chevron·배지 광택 등)는 **`pointerEvents:'none'`**으로 둬 클릭이 아래 요소로 통과되게 한다. 발견성은 살리되 회귀 0. 동작 안내가 필요하면 **첫 1회 nudge 애니메이션**(`localStorage` 플래그로 영구 1회·`prefers-reduced-motion` 제외)으로 시연 — 별도 클릭 핸들 없이 충분.

## D. 데이터 편집 & 검증

### D1. JSX는 `node --check`가 안 되지만 **데이터 배열은 된다**
- **교훈**: 큰 데이터(DAYS/RECOMMENDATIONS 등)를 고치면 `awk`로 **배열 구간만 추출 → `node --check`**로 JS 구문 검증. JSX 본문은 못 하니 **괄호 균형(`grep -o '[(){}]' | sort | uniq -c`)**으로 대체.

### D2. 대량 편집은 "블록 통째 교체"가 안전하다
- **교훈**: 9개 도시 × 12항목처럼 많을 때, 108개 개별 Edit보다 **도시 블록 통째 교체**(8 Edit)가 빠르고 오류 적음. 한 줄=한 객체면 유니크 텍스트로 객체별 Edit.

### D3. 작은따옴표 카운트 홀수 = 꼭 오류는 아니다
- **증상**: 검증 grep에서 작은따옴표 개수 홀수.
- **원인**: 텍스트 내 아포스트로피(`Speaker's`, `All'Antico`)가 큰따옴표로 감싸진 채 섞임.
- **교훈**: 단순 카운트로 패닉하지 말고 **실제 구문검사(node --check)**로 판단. 새로 쓰는 문자열엔 ASCII `'` 대신 한글 인용부호나 큰따옴표를 써 충돌을 피한다.

### D4. 원본은 불변, 편집은 오버레이
- **교훈**: `_DEFAULT` 데이터는 코드에 두고, 사용자 편집은 별도 `edits` 키에 오버레이로 머지. → 앱 업데이트가 사용자 입력을 덮지 않음. 콘텐츠 텍스트를 바꿔도 `id`는 절대 안 바꾼다(storage 키라서).

### D5. 검토 에이전트/자동 리포트는 grep으로 사실 검증한 뒤 쓴다
- **증상**: 디자인 규격 검토를 서브에이전트에 맡겼더니 라인 번호에 "(예상)·(컨텍스트 미확인)"이 섞이고, `borderRadius:50%`를 `--radius-pill`로 바꾸라 권장(원형 아이콘은 50%가 정석), `#fff`/fontFamily 개수도 실제와 어긋남.
- **원인**: 에이전트는 넓게 훑되 라인을 추정·과대 보고할 수 있음.
- **교훈**: 에이전트 리포트는 **방향성 단서로만** 쓰고, 실제 수정 대상은 **`grep -n`으로 개수·위치·컨텍스트를 직접 확정**한 뒤 손댄다. 권장안도 검증(50%↔pill처럼 틀릴 수 있음).

### D6. 모호한 디자인 요청은 *데이터 기준*으로 범위를 확정한다
- **증상**: "도시색이 *없는* 부분을 금빛으로" — 어디가 색이 없는지 불명확.
- **교훈**: 데이터에서 객관 기준을 잡는다 — CHAPTERS에서 무채색(`color:'#F5F5F4'`)은 서막·귀국 둘뿐 → 그게 실제로 *렌더되는 위치*까지 추적(28일 일정 카드 Day1·27·28; ChapterProgress는 p/ret를 필터로 제외해 대상 아님). **"무엇이 비었나"를 코드/데이터로 정의**한 뒤 적용 지점을 좁힌다.

## E. 제품 & 톤 & UX

### E1. 톤은 제품이다 — 독자가 민감하다
- **사례**: "ASI 산물"이라는 라벨이 과하다는 피드백 → `🍪 오늘의 추억 바이트`로. 격식체 → 친근체로 전수 변경 요청.
- **교훈**: 같이 쓰는 사람(독자)의 **취향에 톤을 맞추고, 한 번 정하면 전수 통일**. 톤 규칙을 문서에 박아 이후 모든 콘텐츠에 일괄 적용.

### E2. "공유 앱" 전환은 콘텐츠 규칙을 바꾼다
- **사례**: 1인용 → 배우자 공유로 전제가 바뀌자, 개인 커리어·학교·투자 용어(GSB/LP/회사)를 보편·공동 톤으로 교체.
- **교훈**: **"누가 읽는가"가 바뀌면 콘텐츠 전수 점검.** 공개 화면은 공동 톤, PIN(Private) 영역만 1인칭 사색 허용 — 영역별로 톤을 분리.

### E3. 깊이는 "한 스푼"씩, 접어서 둔다
- **교훈**: 맥락(why)·심화질문(qchain)은 좋지만 항상 펼쳐두면 화면이 무겁다. **접힘 토글**로 평소 깔끔, 원할 때만 깊이. 철학/역사/경제는 과하지 않게 한 스푼.

### E4. 파일럿 → 확장으로 톤을 합의한다
- **교훈**: 28일 전체를 한 번에 쓰지 말고 **1챕터(파리 3일) 먼저** 만들어 톤·깊이 합의 → OK 후 나머지. 손 되돌리는 비용을 줄인다.

### E5. 강조 장식의 "확장"은 의미색·과용을 경계한다
- **사례**: 무채 챕터 카드의 금빛 글로시(`gt-gold-edge`)가 반응이 좋아 4곳으로 확장(Lifetime 카드·헤더 라인·하단네비 활성탭·확정 배지). 단 확정 배지의 기존 의미색(초록=확정)을 골드로 바꾸면 status 색 체계(🔴긴급/🟡미예약/✅확정)가 붕괴되고, 배지가 다수라 골드가 과해짐.
- **교훈**: 좋은 효과라도 **의미색(상태·카테고리 색)은 보존**하고, 강조는 *겹쳐 쓰는* 방식(테두리 광택 `gt-gold-glow` 정적 glow)으로 더한다. quiet luxury = "특별한 곳에만". 적용 전 **"여기 골드가 의미적으로 정당한가 / 몇 개나 동시에 보이나"**를 묻는다. 큰 요소엔 sheen(흐름 애니메이션), 작은·다수 요소엔 정적 glow로 강도를 나눈다. 모든 sheen은 `prefers-reduced-motion`에서 정지.

### E6. 가로 스크롤 셀렉터는 "숨은 선택지"다 — 발견성 vs 가로 유지의 기준
- **사례**: 보조 셀렉터 5곳(번역/가계부 카테고리·음악/날씨/복장 필터)이 가로 스크롤(`gt-chips`)이라 선택지 일부가 화면 밖에 숨어 있었음 → 줄바꿈(`.gt-wrap` = `flex-wrap`)으로 전부 노출. 단 28일 타임라인(28개)·일정 챕터 필터(라벨 긴 챕터)는 가로 유지.
- **교훈**: 가로 스크롤은 "더 있다"는 신호가 약해 **선택지를 숨긴다**. **한 화면에 들어갈 만큼(항목 수·라벨 길이)이면 줄바꿈이 우월**(모든 옵션이 한눈에). 가로 유지의 정당한 기준은 *한 화면 초과* 하나뿐. Logistics 서브탭을 grid로 푼 것과 같은 철학. **레이아웃 클래스를 바꿀 땐 기존 인라인(`justifyContent:'center'` 등)과 충돌하는지 확인** — 충돌하면 컨테이너는 두고 자식(버튼)만 손댄다.

### E7. "토큰 통일"은 값을 한 곳에 모아야 진짜다
- **사례**: 칩/탭 전환 모션(`transition: ... 0.15s`)이 여러 컴포넌트에 **인라인으로 복붙**돼 있었음. 동일 문자열이 분산되면 "토큰화"라 부를 수 없음 → `.gt-chip` CSS 클래스 1곳으로 모으고 인라인 transition은 전부 제거(잔여 0).
- **교훈**: 같은 값을 N곳에 인라인으로 두면 토큰의 이점(일괄 변경·일관성)이 0. **단일 클래스/변수로 모아 className만 부여**. grep으로 "인라인 잔여 0"을 검증 기준에 넣는다. 토큰 이름≠값(C5)·토큰화≠무변화(C6) 교훈과 한 묶음.

### E8. 토큰화 ≠ 스냅 — 두 스케일 공존은 "exact-value 토큰 + alias"로 봉합한다
- **사례**: radius가 t-shirt 토큰(4/6/10/14/18/24, `var()` 143곳)과 코드 실사용 4배수값(8/12/16/20, 인라인 41곳)으로 **이원화**. 13차는 "8px를 6/10으로 스냅하면 1~4px 회귀"라 보류. 14차 해법 = 스케일을 numeric(px=이름) **단일 진실원천**으로 재정의하고 인라인을 그 토큰으로 치환하되 **값은 그대로**, 기존 t-shirt 이름은 `--radius-xs: var(--radius-4)`처럼 **alias**로 둬 호출부 143곳을 안 건드림.
- **교훈**: "안전 정리"의 정의 = **값 보존**. 토큰이 없어 보이면 *스냅*(가까운 토큰으로 끌어당김=회귀)이 아니라 *exact-value 토큰을 추가*한다(회귀 0 + 구조적 불일치 해소). 두 명명 체계가 충돌하면 한쪽을 **alias로 다른 쪽에 종속**시켜 단일 진실원천을 세우는 무위험 마이그레이션을 택한다(전수 rename churn 회피). px↔em 환산이 끼는 값(자간)은 폰트크기 의존이라 토큰화 자체가 회귀 — 의도된 잔여로 두고 손대지 않는다(C6 연장).

## G. 한 줄 요약 (다음 프로젝트 시작 전 읽기)

1. **반영 안 되면** 디스크→커밋→push→배포→SW 체인을 끝까지 확인.
2. **외부 자원은 죽는다** — 항상 폴백.
3. **데이터 배열은 node --check**, JSX는 괄호 균형으로 검증.
4. **톤은 제품** — 독자에 맞춰 전수 통일, 문서에 박기.
5. **파일럿 먼저**, 확인 후 확장.
6. **원본 불변 + edits 오버레이**, id는 절대 안 바꾼다.
7. **push는 오너 수동**, 시각 변경은 확인 후 커밋.
8. **Desktop은 TCC 지뢰밭** — 비보호 폴더 권장, 헤드리스 금지.
9. **토큰 이름 ≠ 값** — 정의를 grep으로 확인하고, "토큰화=무변화"는 상위 오버라이드·px↔em 환산까지 따져 가정 금지.
10. **에이전트 리포트는 grep으로 사실 검증** 후 손댄다. 모호한 요청은 **데이터 기준으로 범위 확정**.
11. **동기 script가 첫 페인트를 막는다** — `defer`로 분리(babel-standalone은 DOMContentLoaded 변환이라 호환), 단 흰화면 실기기 확인.
12. **실시간 동기화는 편집 입력을 덮는다** — `baseRef`+dirty로 편집 중이면 보존 + 충돌 배너(LWW).
13. **강조 장식 확장은 의미색 보존·과용 경계** — "특별한 곳에만", 의미색 위엔 겹쳐 쓰기. 오버레이 시각신호는 `pointerEvents:'none'`로 탭 무회귀.
14. **가로 스크롤은 선택지를 숨긴다** — 한 화면에 들어가면 줄바꿈(`flex-wrap`)이 우월, 가로 유지는 한 화면 초과일 때만. **토큰 통일은 값을 단일 클래스/변수로 모아야 진짜**(인라인 잔여 0을 grep으로 검증).
15. **토큰화는 스냅이 아니다** — 토큰 없는 값은 가까운 토큰으로 끌어당기지(회귀) 말고 **exact-value 토큰을 추가**한다(값 보존=회귀 0). 두 스케일이 충돌하면 한쪽을 **alias로 종속**시켜 호출부 전수 rename 없이 단일 진실원천을 세운다. px↔em 환산 값(자간)은 토큰화 자체가 회귀라 의도된 잔여로 둔다.
