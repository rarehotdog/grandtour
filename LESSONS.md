## A. 배포 & 캐시

### A1. "코드 고쳤는데 폰 앱에 반영이 안 돼요"
- **증상**: 로컬에서 다 됐는데 라이브 PWA는 옛날 화면.
- **원인**: ① 커밋만 하고 `git push`를 안 함 → GitHub/Vercel은 옛 버전. ② push해도 PWA가 Service Worker 캐시로 옛 HTML 서빙.
- **교훈**: 반영 안 될 때 **체인을 끝까지 확인**: `디스크 파일 → 커밋 → push → Vercel 배포 → SW 캐시`. 어디서 끊겼는지부터 짚는다. "보는 화면이 file://인지 라이브 URL인지"를 가장 먼저 확정.

### A2. SW 캐시가 끈질기다
- **원인**: `network-first HTML`이라도 PWA standalone은 갱신 타이밍이 느릴 수 있음.
- **교훈**: HTML만 바꿨으면 PWA **완전 종료 후 재실행**이면 보통 픽업. 그래도 안 되면 Safari 웹사이트 데이터 삭제. **`sw.js` 자체를 고쳤을 때만** `CACHE` 버전 상수를 bump한다(콘텐츠만 바꿨는데 bump하면 불필요한 전면 재캐시).

### A3. push는 함부로 하지 않는다
- **교훈**: push = 외부 배포(되돌리기 번거롭고 outward-facing). **오너가 명시 요청할 때만** 실행하고, 평소엔 "푸시 직전까지"만. 커밋까지는 자유, push는 확인 후.

## B. 환경 & 권한 (macOS / Desktop)

### B1. `~/Desktop` 트리의 TCC / `com.apple.macl` 잠금
- **증상**: 파일을 브라우저/에디터로 연 뒤 갑자기 Claude의 파일 read/write가 전부 `Operation not permitted`.
- **원인**: macOS TCC 보호 폴더에서 파일을 GUI 앱이 열면 `com.apple.macl` xattr가 붙어 다른 프로세스 접근 차단.
- **교훈**: 프로젝트는 가능하면 **`~/Develop/` 등 비보호 폴더**에 둔다. Desktop이면: 파일 연 앱 닫기 + Claude 재시작, 또는 터미널에 Full Disk Access 부여.

### B2. Desktop에서 헤드리스 Chrome 금지
- **원인**: 헤드리스 크롬 구동이 위 TCC 잠금을 유발.
- **교훈**: 시각 검증은 **헤드리스 대신** ① 텍스트/구조 검증(grep·구문검사) ② 일반 Safari `open`(오너가 직접 봄)으로 대체.

## C. 코드 & Babel & CSS

### C1. JSX Fragment `<>` 가 안 먹는다
- **원인**: Babel standalone(브라우저 런타임 컴파일)이 일부 fragment 문법 처리 못 함.
- **교훈**: `<div>` 또는 `React.Fragment` 사용. 단일 파일 = Babel standalone 전제이므로 항상.

### C2. 미정의 CSS 변수를 써서 스타일이 조용히 사라짐
- **증상**: `border: 1px dashed var(--color-hairline)` 인데 선이 안 보임(에러도 없음).
- **원인**: 존재하지 않는 토큰명. CSS 변수는 미정의여도 에러 없이 무시됨 → 디버깅 어려움.
- **교훈**: 토큰 쓰기 전 **`grep -- "--토큰명:"`로 정의 존재 확인**. 작업 후 `grep "var(--…)"` 대비 정의 목록 대조.

### C3. SMIL 애니메이션은 살아있다(iOS Safari)
- **교훈**: `<animateMotion>`/`<animate>`(SMIL)은 React JSX에서 그대로 렌더되고 iOS Safari가 지원 → **의존성 0으로 경로 애니메이션** 가능. 단 `prefers-reduced-motion`은 SMIL을 CSS로 못 끄니, draw류는 CSS keyframes로 따로 빼서 reduced 처리.

### C4. 외부 이미지·폰트는 반드시 죽는다고 가정
- **증상**: Unsplash 직링크가 깨져 검정 박스.
- **교훈**: 모든 외부 이미지에 **`onError` → 그래디언트/폴백**. 폰트도 시스템 폰트 폴백. "외부 자원은 언젠가 죽는다"를 기본값으로.

## D. 데이터 편집 & 검증

### D1. JSX는 `node --check`가 안 되지만 **데이터 배열은 된다**
- **교훈**: 큰 데이터(DAYS/RECOMMENDATIONS 등)를 고치면 `awk`로 **배열 구간만 추출 → `node --check`**로 JS 구문 검증. JSX 본문은 못 하니 **괄호 균형(`grep -o '[(){}]' | sort | uniq -c`)**으로 대체.

### D2. 대량 편집은 "블록 통째 교체"가 안전하다
- **교훈**: 9개 도시 × 12항목처럼 많을 때, 108개 개별 Edit보다 **도시 블록 통째 교체**(8 Edit)가 빠르고 오류 적음. 한 줄=한 객체면 유니크 텍스트로 객체별 Edit.

### D3. 작은따옴표 카운트 홀수 = 꼭 오류는 아니다
- **증상**: 검증 grep에서 작은따옴표 개수 홀수.
- **원인**: 텍스트 내 아포스트로피(`Speaker's`, `All'Antico`)가 큰따옴표로 감싸진 채 섞임.
- **교훈**: 단순 카운트로 패닉하지 말고 **실제 구문검사(node --check)**로 판단. 새로 쓰는 문자열엔 ASCII `'` 대신 한글 인용부호나 큰따옴표를 써 충돌을 피한다.

### D4. 원본은 불변, 편집은 오버레이
- **교훈**: `_DEFAULT` 데이터는 코드에 두고, 사용자 편집은 별도 `edits` 키에 오버레이로 머지. → 앱 업데이트가 사용자 입력을 덮지 않음. 콘텐츠 텍스트를 바꿔도 `id`는 절대 안 바꾼다(storage 키라서).

## E. 제품 & 톤 & UX

### E1. 톤은 제품이다 — 독자가 민감하다
- **사례**: "ASI 산물"이라는 라벨이 과하다는 피드백 → `🍪 오늘의 추억 바이트`로. 격식체 → 친근체로 전수 변경 요청.
- **교훈**: 같이 쓰는 사람(독자)의 **취향에 톤을 맞추고, 한 번 정하면 전수 통일**. 톤 규칙을 문서에 박아 이후 모든 콘텐츠에 일괄 적용.

### E2. "공유 앱" 전환은 콘텐츠 규칙을 바꾼다
- **사례**: 1인용 → 배우자 공유로 전제가 바뀌자, 개인 커리어·학교·투자 용어(GSB/LP/회사)를 보편·공동 톤으로 교체.
- **교훈**: **"누가 읽는가"가 바뀌면 콘텐츠 전수 점검.** 공개 화면은 공동 톤, PIN(Private) 영역만 1인칭 사색 허용 — 영역별로 톤을 분리.

### E3. 깊이는 "한 스푼"씩, 접어서 둔다
- **교훈**: 맥락(why)·심화질문(qchain)은 좋지만 항상 펼쳐두면 화면이 무겁다. **접힘 토글**로 평소 깔끔, 원할 때만 깊이. 철학/역사/경제는 과하지 않게 한 스푼.

### E4. 파일럿 → 확장으로 톤을 합의한다
- **교훈**: 28일 전체를 한 번에 쓰지 말고 **1챕터(파리 3일) 먼저** 만들어 톤·깊이 합의 → OK 후 나머지. 손 되돌리는 비용을 줄인다.

## F. 협업 & Git & 문서

### F1. REVIEW.md를 ground truth로 유지
- **교훈**: 매 작업 후 REVIEW에 ① 현 상태 표 ② 커밋 이력 ③ **다음 개선 통합 우선순위 표** 갱신. 다음 세션이 REVIEW + git log만 보고 이어갈 수 있게. 우선순위 기준 문장을 고정(*"5초 효용 + 실패 시 타격"*).

### F2. 커밋 단위는 "시각 확인이 필요한 것"으로 끊는다
- **교훈**: 데이터/톤 변경은 검증 후 바로 커밋. **애니메이션·레이아웃 등 눈으로 봐야 하는 변경은 시각 확인 후 커밋**(미커밋 상태로 두고 확인 요청). 깨질 수 있는 것과 안전한 것을 분리.

### F3. 메모리에 "재발 방지 사실"을 남긴다
- **교훈**: TCC 잠금, 톤 규칙, 동기화 제외 키처럼 **다음 세션이 모르면 사고나는 것**은 프로젝트 메모리에 박는다. 코드/깃에 이미 있는 건 중복 저장 안 함.

## G. 한 줄 요약 (다음 프로젝트 시작 전 읽기)

1. **반영 안 되면** 디스크→커밋→push→배포→SW 체인을 끝까지 확인.
2. **외부 자원은 죽는다** — 항상 폴백.
3. **데이터 배열은 node --check**, JSX는 괄호 균형으로 검증.
4. **톤은 제품** — 독자에 맞춰 전수 통일, 문서에 박기.
5. **파일럿 먼저**, 확인 후 확장.
6. **원본 불변 + edits 오버레이**, id는 절대 안 바꾼다.
7. **push는 오너 수동**, 시각 변경은 확인 후 커밋.
8. **Desktop은 TCC 지뢰밭** — 비보호 폴더 권장, 헤드리스 금지.
